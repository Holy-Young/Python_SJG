❓ ① Hello World 常见问题解答

上节课写完第一个程序，你可能有很多疑问。我们来逐一解答：

🤔 为什么 "Hello, World!" 要加引号？

在 Python 里，引号里的内容叫做"字符串"，表示"这是一段文字"。如果不加引号，Python 会以为 Hello 是某个变量名，找不到就报错。

✅ 正确写法
print("Hello, World!")
❌ 错误写法
print(Hello, World!)
NameError: name 'Hello' is not defined
🔤 print 能不能写成 Print 或 PRINT？

不能！Python 是大小写敏感的语言，print、Print、PRINT 是三个完全不同的东西。只有 print（全小写）才是 Python 内置的输出函数。

✅ 正确
print("你好")
❌ 错误
Print("你好")
NameError: name 'Print' is not defined
🔢 括号是必须的吗？能用中文括号吗？

括号是必须的，它告诉 Python "这里是在调用一个函数"。并且括号必须是英文括号 ()，中文括号 （） 会导致语法错误。

💡
小技巧

在写代码时，把输入法切换到英文模式，所有的符号（括号、引号、逗号）都用英文，可以避免大部分这类错误。

🈳 为什么有时候打印中文会乱码或报错？

现代 Python 3 默认使用 UTF-8 编码，支持中文，一般不会乱码。但如果你的代码文件保存时选了错误的编码格式（比如 GBK），就可能出问题。

解决方法：在代码文件第一行加上这句声明：

Python 
# -*- coding: utf-8 -*-
print("你好，世界！")
🔴 看到红色报错信息怎么办？

报错信息是 Python 在帮你找 bug，不要怕它！学会读懂报错信息是非常重要的技能。常见错误类型：

错误类型
含义
常见原因
SyntaxError
语法错误
括号/引号没闭合，用了中文符号
NameError
名称错误
变量/函数名拼错，或大小写不对
IndentationError
缩进错误
代码前面的空格数量不对
TypeError

数据类型用错了，如数字和字符串混用
💬 ② 什么是编程语言？

你有没有想过，电脑其实只懂 0 和 1？所有的程序，最终都要变成一串 0 和 1 才能让 CPU 执行。而编程语言，就是人类和电脑沟通的"翻译工具"。

编程语言的进化史

1
第一代 · 1940s

机器语言

直接用 0 和 1 写代码，电脑看得懂，但人类几乎不可能直接阅读和书写。

机器语言 
10110000 01100001   ← 这是"把 97 存入寄存器"的意思
2
第二代 · 1950s

汇编语言

用助记符（英文缩写）代替 0 和 1，稍微人性化一点，但仍然复杂，而且不同 CPU 要写不同的代码。

汇编语言 
MOV AL, 61h    ← 同样是"把 97 存入寄存器"
3
第三代 · 1960s至今

高级语言 ← 我们在这里！

接近人类自然语言，一份代码可以在多种设备上运行。Python、Java、C++ 都属于高级语言。

Python（高级语言） 
print("Hello!")   ← 简单易懂，人人都能读懂

⚙️ ③ Python 解释器是什么？

我们写的 Python 代码，电脑并不能直接看懂。需要有一个"翻译官"把代码翻译成电脑能理解的指令——这个翻译官就叫做 Python 解释器。

解释器 vs 编译器

把代码翻译给电脑有两种方式，就像两种不同的翻译工作：

🔄 解释器（Python 用这种）

像"同声传译"——边读代码边翻译边执行，一行一行来。不需要等全部翻译完，马上就能看到结果。

源代码
→
解释器逐行翻译
→
立即执行
✅ 调试方便，出错马上知道
✅ 跨平台好，换个系统装解释器就能运行
⚠️ 运行速度相对慢
📦 编译器（C/C++ 用这种）

像"提前翻译好的书"——先把所有代码一次性翻译成机器码，保存成可执行文件，之后每次直接运行翻译结果。

源代码
→
编译器全部翻译
→
生成 .exe 文件
✅ 运行速度极快
⚠️ 编译时间长，跨平台需要重新编译

最常用的解释器：CPython

🏆
CPython 是什么？

CPython 是 Python 官方提供的解释器，用 C 语言编写，也是使用最广泛的版本。我们从官网下载安装的就是 CPython。你在命令行输入 python 启动的，也是它。

Python 的两种运行方式

💬
交互模式
适合练习和测试

打开命令行输入 python 回车，会看到 >>> 提示符，输入一行代码马上看到结果。

命令行 · 交互模式 
>>> print("Hello!")
Hello!
>>> 1 + 2
3
>>>
📄
脚本模式
适合写完整程序

把代码写进 .py 文件，用命令行运行 python 文件名.py，一次执行所有代码。

命令行 · 脚本模式 
C:\Users\你> python hello.py
Hello, World!
你好，Python！

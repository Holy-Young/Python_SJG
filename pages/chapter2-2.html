📦 ① 变量的概念

在上一节我们学习了字面量——那些直接写在代码里的固定值。但是,如果我们想让程序"记住"一些数据,或者让数据可以改变,就需要用到变量(Variable)。

🎒
生活中的例子

你可以把变量想象成一个带标签的盒子。盒子上写着名字(变量名),盒子里装着东西(变量的值)。你随时可以往盒子里放新东西,或者拿出来看看里面装的是什么。

比如:你有一个叫"书包"的盒子,今天里面装的是数学课本,明天可以换成语文课本。盒子的名字不变,但里面的内容可以改变!

变量 vs 字面量

让我们通过对比来理解变量和字面量的区别:

💎

字面量

  • 固定不变的值
  • 直接写在代码中
  • 如: 100, "你好", 3.14
  • 不能被修改
📦

变量

  • 可以改变的值
  • 有一个名字(标识符)
  • 如: age, name, score
  • 可以被修改和重新赋值

为什么需要变量?

想象一下,如果没有变量会怎么样:

❌ 没有变量的写法 
# 计算一个学生的总分和平均分
print("语文:", 85)
print("数学:", 92)
print("英语:", 88)
print("总分:", 85 + 92 + 88)
print("平均分:", (85 + 92 + 88) / 3)

# 如果分数改了,要改很多地方!太麻烦了!
✅ 有变量的写法 
# 用变量存储分数
chinese = 85
math = 92
english = 88

# 计算总分和平均分
total = chinese + math + english
average = total / 3

print("语文:", chinese)
print("数学:", math)
print("英语:", english)
print("总分:", total)
print("平均分:", average)

# 分数改了,只需要改一个地方!
✨
变量的好处
  • 方便修改: 只需要改变量的值,不用到处修改代码
  • 代码可读: 看到变量名就知道它代表什么意思
  • 复用数据: 同一个数据可以在多个地方使用
  • 程序灵活: 可以让数据在程序运行中动态变化
🎯 ② 变量的使用

定义变量和赋值

在 Python 中,定义变量非常简单,只需要用 = 把值赋给一个名字就可以了。这个过程叫做赋值(Assignment)。

赋值操作演示

age = 14
→
创建变量 age, 存入值 14
name = "小明"
→
创建变量 name, 存入值 "小明"
is_student = True
→
创建变量 is_student, 存入值 True
⚠️
重要提醒

在 Python 中,= 不是"等于"的意思,而是赋值的意思!它的作用是把右边的值存入左边的变量中。

所以 age = 14 读作"把 14 赋值给 age",而不是"age 等于 14"。

💻 变量的基本操作 
# 1. 定义变量
name = "小红"
age = 13
height = 1.58
is_student = True

# 2. 使用变量
print("姓名:", name)
print("年龄:", age)
print("身高:", height, "米")
print("是学生吗?", is_student)

# 3. 修改变量的值
age = 14  # 过了一年,年龄变了
height = 1.62  # 长高了!

print("
一年后...")
print("年龄:", age)
print("身高:", height, "米")
▶️ 运行结果
姓名: 小红 年龄: 13 身高: 1.58 米 是学生吗? True 一年后... 年龄: 14 身高: 1.62 米

🎯

使用前必须赋值

不能使用没有定义的变量,会报错!

📝 ③ 标识符规则

标识符(Identifier)就是我们给变量、函数、类等起的名字。在 Python 中,标识符的命名有一些规则,必须遵守,否则程序会报错!

标识符的命名规则 📏

⚡
必须遵守的硬性规则(违反会报错!)
  • 只能包含: 字母(a-z, A-Z)、数字(0-9)、下划线(_)
  • 不能以数字开头: 比如 1name 是错误的
  • 不能使用关键字: 比如 if, for, print 等是保留字
  • 大小写敏感: Name 和 name 是两个不同的变量
  • 不能有空格: 不能写成 my name,要写成 my_name
规则说明 ✅ 正确示例 ❌ 错误示例 错误原因
字母开头 name, age, score 1name, 2age 不能以数字开头
可以用下划线 _name, my_age, __private my-age, my age 不能用连字符或空格
可以包含数字 score1, student2, var_123 1score, 2student 数字不能在开头
大小写敏感 Name, name, NAME (这三个是不同的变量) Python区分大小写
不能用关键字 my_if, for_loop, class_name if, for, class 这些是Python保留字
支持中文(不推荐) 名字, 年龄 (虽然可以,但不推荐) 国际化代码应用英文

小提示

虽然 Python 在技术上支持使用中文、日文等作为变量名,但在实际编程中强烈不推荐这样做!

原因: ① 团队协作困难 ② 在某些系统可能有编码问题 ③ 不符合国际编程规范

❌ 错误的命名示例 
# 这些都是错误的命名,会导致程序报错!

1name = "小明"        # ❌ 不能以数字开头
my-age = 14          # ❌ 不能使用连字符
my name = "李华"     # ❌ 不能包含空格
for = 10             # ❌ for 是关键字
class = "三年二班"   # ❌ class 是关键字
@score = 95          # ❌ 不能使用特殊符号
✅ 正确的命名示例 
# 这些都是正确的命名

name1 = "小明"           # ✅ 可以包含数字(但不能开头)
my_age = 14             # ✅ 使用下划线连接
myName = "李华"         # ✅ 驼峰命名法
student_score = 95      # ✅ 见名知意
_private_data = "隐藏"   # ✅ 下划线开头(通常表示私有)
MAX_VALUE = 100         # ✅ 全大写(通常表示常量)
for_loop_count = 5      # ✅ 避开关键字
class_name = "三年二班"  # ✅ 避开关键字
✨ ④ 命名建议

除了必须遵守的规则外,还有一些最佳实践和约定俗成的规范。虽然不遵守不会报错,但遵守这些规范能让你的代码更专业、更易读!

命名风格 🎨

🐍 下划线命名法 (snake_case)

推荐用于: 变量名、函数名

student_name = "小明"
total_score = 285
is_pass = True
max_retry_count = 3

✅ 特点: 单词之间用下划线分隔,全部小写,Python 官方推荐!

🐫 小驼峰命名法 (camelCase)

推荐用于: 部分变量(看团队习惯)

studentName = "小明"
totalScore = 285
isPass = True
maxRetryCount = 3

✅ 特点: 第一个单词小写,后续单词首字母大写

🦒 大驼峰命名法 (PascalCase)

推荐用于: 类名

StudentInfo
ScoreCalculator
DataManager
GameController

✅ 特点: 每个单词首字母都大写(后面学类的时候会用到)

📢 全大写命名法 (UPPER_CASE)

推荐用于: 常量

MAX_SCORE = 100
PI = 3.14159
DEFAULT_NAME = "未命名"
RETRY_LIMIT = 3

✅ 特点: 全部大写,下划线分隔,表示这个值不应该被修改

见名知意原则 👁️

好的变量名应该让人一眼就能看出它存储的是什么数据。这是编程中最重要的原则之一!

❌ 不好的命名

a = "小明"       # 看不出是什么
b = 14          # 完全不知道含义
x = 95          # x 代表什么?
n = "三年二班"   # n 太模糊了
temp = 36.5     # temp 可能是温度,也可能是临时变量

✅ 好的命名

student_name = "小明"      # 一看就知道是学生姓名
age = 14                  # 清楚表示年龄
math_score = 95           # 明确是数学分数
class_name = "三年二班"    # 清楚表示班级名称
body_temperature = 36.5   # 明确是体温

其他命名建议 💡

1️⃣
避免使用单个字母

除非是在很短的代码块中做临时变量(如循环中的 i, j),否则不要用 a, b, x, y 这种名字

2️⃣
长度适中

不要太短(n),也不要太长(this_is_a_very_long_variable_name_that_stores_student_score)。2-4个单词最合适

3️⃣
使用英文

虽然 Python 支持中文变量名,但在正式项目中应该用英文。如果英文不好,可以用翻译工具帮忙

4️⃣
布尔变量用 is/has 开头

表示真假的变量,通常用 is_, has_, can_ 开头,如 is_student, has_homework, can_swim

5️⃣
保持一致性

在同一个项目中,应该保持命名风格一致。如果用下划线法,就全部用下划线法;如果用驼峰法,就全部用驼峰法

🌟 完整的命名示例 
# 学生信息管理示例 - 展示好的命名习惯

# 常量(全大写)
MAX_STUDENTS = 50
PASS_SCORE = 60

# 学生基本信息(下划线命名法)
student_name = "张三"
student_age = 14
student_gender = "男"
student_id = "2024001"

# 成绩信息
chinese_score = 85
math_score = 92
english_score = 88
total_score = chinese_score + math_score + english_score
average_score = total_score / 3

# 布尔值(is开头)
is_pass = average_score >= PASS_SCORE
is_excellent = average_score >= 90
has_homework = True
can_graduate = True

# 输出结果
print(f"学生姓名: {student_name}")
print(f"总分: {total_score}")
print(f"平均分: {average_score:.2f}")
print(f"是否及格: {is_pass}")
print(f"是否优秀: {is_excellent}")
🎯
命名的黄金法则

让别人(或三个月后的自己)能立即看懂你的代码!

好的命名就像给代码贴上了清晰的标签,让程序的逻辑一目了然。花点时间想一个好名字,比写一堆注释解释要好得多!
